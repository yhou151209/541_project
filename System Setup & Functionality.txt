Overall Setup
The system schedules restaurant staff across 7 days and 2 shifts per day (morning & evening).
	Each shift has predefined role requirements (cashier, server).
	Employees have:
		Skills (e.g., server, cashier)
		Availability
		Maximum number of shifts
		Employment type (used as a proxy for experience)


Core Constraints (Hard Constraints)
The schedule must always satisfy:
	Each shift is fully covered (required number of staff per role)
	Employees are only assigned if:
		They are available
		They have the required skills
	Each employee can work at most one role per shift
	Each employee respects their maximum shift limit


Optimization Objectives (Soft Constraints)
The system optimizes the schedule by:
	Minimizing changes from the existing schedule (stability)
	Balancing workload across employees
	Reducing violations of user preferences
	Prioritizing full-time staff on busy shifts
		Busy shifts include:
			All evening shifts
			All weekend shifts (Saturday & Sunday)


Supported Update Types
The system supports four types of updates:
	1.Unavailable
		Removes an employee from a specific shift
		Automatically reassigns replacements
	2.Direct Swap
		Swaps two employees’ shifts (if feasible)
		Ensures skill compatibility and constraint satisfaction
	3.Avoid Back-to-Back
		Preference to avoid assigning both morning and evening on the same day
	4.Avoid Shift (Flexible Granularity)
		Avoid a shift type (e.g., evening)
		Avoid a specific day (e.g., Sunday)
		Avoid a specific day + shift (e.g., Saturday evening)
		
		
Explanation Module
The system provides rule-based explanations for each update:
	Why assignments changed
	Why certain employees were selected
	How preferences influenced the result
Includes reasoning such as:
	Availability constraints
	Skill matching
	Busy shift prioritization
	Minimal change principle
